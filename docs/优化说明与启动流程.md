# 优化说明与启动流程（2026-03）

## 本次优化内容

### 1) 代码层优化
- 新增全局异常处理：`src/main/java/com/seckill/exception/GlobalExceptionHandler.java`
- 对参数校验异常、请求体格式异常、通用异常做统一返回，避免接口直接抛出 400/500 默认页面。
- `SearchController` 的 `/api/search/sync` 返回文案调整为“未启用 ES 时会自动跳过”，避免误导。
- 新增数据库自动初始化脚本：
  - `src/main/resources/db/init_user.sql`（自动建 `user` 表）
  - `src/main/resources/db/init_seckill_product.sql`（自动建商品表）
  - `src/main/resources/db/init_seckill_product_data.sql`（仅首次空表导入商品数据）

### 2) 启动脚本优化
- 重构 `run-without-es.ps1`，去掉硬编码绝对路径，支持在任意机器上直接运行。
- 增加参数：
  - `-SkipBuild`：仅启动，不执行构建
  - `-SkipTests`：构建时跳过测试
- 自动检测 `mvnw.cmd` 是否存在，错误提示更明确。
- 统一使用项目本地 Maven 仓库（`.m2repo`），减少环境差异问题。

### 3) 启动流程优化
- 统一推荐“先依赖、后后端”的最短路径：
  1. 启动 MySQL + Redis
  2. 初始化数据库
  3. 执行 `./run-without-es.ps1 -SkipTests`
- 默认建议使用 `no-es` profile 开发，降低本地依赖复杂度。

## 快速启动（Windows / PowerShell）

在项目根目录执行：

```powershell
./run-without-es.ps1 -SkipTests
```

若你已经构建过，想直接启动：

```powershell
./run-without-es.ps1 -SkipBuild
```

启动成功后访问：
- 接口基地址：`http://localhost:8083`
- 商品列表：`http://localhost:8083/api/products/list`

## 自动初始化说明

- 已开启 `spring.sql.init.mode=always`。
- 已配置：
  - `spring.sql.init.schema-locations=classpath:db/init_user.sql,classpath:db/init_seckill_product.sql`
  - `spring.sql.init.data-locations=classpath:db/init_seckill_product_data.sql`
- 启动应用时会自动建表；商品数据只会在 `seckill_product` 为空时导入一次。
- 每次启动都会写入一条审计记录到 `startup_init_audit`，用于确认本次启动导入状态。

## 初始化健康检查

接口：`GET /api/health/init-status`

用于确认“本次启动是否触发首次导入”，关键字段：
- `seededThisStartup`：`true` 表示本次启动触发了首次导入
- `productCount`：当前 `seckill_product` 记录数
- `latestSeedAudit`：最近一次导入审计（`seeded` + `createTime`）

## 常见问题排查

### 1) 启动时报数据库连接失败
- 检查 `src/main/resources/application.yml` 的 MySQL 地址/端口/账号密码。
- 若使用 `docker-compose`，主机直连 MySQL 端口通常是 `3307`。

### 2) Redis 连接失败
- 检查 Redis 是否监听 `6379`。
- 检查 `spring.data.redis.host/port` 配置。

### 3) 搜索接口返回空结果
- 默认 `elasticsearch.enabled=false`，为预期行为。
- 如需启用，请在配置中打开 ES 并执行 `POST /api/search/sync`。
