# 分布式功能实现总结

本文档描述的是“当前代码仓库真实实现状态”，重点区分：
- 代码已实现
- 默认配置是否启用
- 落地使用前还需补充什么

## 1. 功能实现矩阵

| 模块 | 代码状态 | 默认状态 | 说明 |
|---|---|---|---|
| 用户注册/登录 | 已实现 | 启用 | 接口：`/api/users/register`、`/api/users/login` |
| 商品列表/详情/秒杀 | 已实现 | 启用 | 接口：`/api/products/list`、`/api/products/{id}`、`/api/products/seckill/{id}` |
| Redis 缓存 | 已实现 | 启用 | 空值缓存、互斥锁重试、随机 TTL |
| Nginx 负载均衡 | 已提供配置 | 需手工部署 | `nginx.conf` 提供轮询/权重/IP Hash/最少连接/URL Hash |
| 动静分离 | 已提供配置 | 需手工部署 | Nginx 直接托管 `src/main/resources/static` |
| Elasticsearch 搜索 | 已实现 | 默认关闭 | 默认 `elasticsearch.enabled=false`，关闭时搜索返回空分页 |
| MySQL 读写分离切换 | 已实现（切面 + 注解） | 默认未生效 | 缺少 `spring.datasource.dynamic.*` 主从配置，需补齐后启用 |

## 2. 关键实现说明

### 2.1 Redis 缓存策略
核心类：`CacheServiceImpl`

- 缓存 key：`seckill:product:{id}`
- 空值缓存：`__NULL__`，TTL 30 秒
- 基础 TTL：30 分钟 + 0~10 分钟随机值
- 并发保护：`ReentrantLock + 重试读取`

### 2.2 秒杀扣减逻辑
核心类：`SeckillProductServiceImpl`

- 使用数据库原子更新：`stock = stock - 1 and stock > 0`
- 秒杀成功后删除缓存，走 Cache Aside
- 库存不足和商品不存在分别返回不同错误信息

### 2.3 搜索模块降级行为
核心类：`SearchServiceImpl`

- 仓库注入使用 `@Autowired(required = false)`
- 当 ES 仓库不可用时：
  - 查询接口返回空分页
  - 同步接口直接跳过（日志给出 warning）

## 3. 与文档最容易混淆的点

1. 商品接口不是 `/api/products`，而是 `/api/products/list`。
2. 秒杀接口不是 `/api/seckill/{id}`，而是 `/api/products/seckill/{id}`。
3. 默认端口是 `8083`，只有 `instance1/instance2` profile 才是 `8081/8082`。
4. 默认配置并没有完整启用主从动态数据源。
5. 初始化 SQL 仅包含商品表，`user` 表需要手动建。

## 4. 建议的落地顺序

1. 先跑通单实例（8083）+ MySQL + Redis。
2. 补齐 `user` 表后验证注册/登录。
3. 再启动双实例（8081/8082）+ Nginx 验证负载均衡。
4. 最后按需启用 Elasticsearch 与读写分离。

## 5. 后续可演进方向

- 登录鉴权：从“返回字符串 token”升级为 JWT + 拦截器。
- 分布式锁：从本地 `ReentrantLock` 升级为 Redisson。
- ES 同步：从手工触发升级为 Binlog/消息驱动增量同步。
- 可观测性：增加 Prometheus + Grafana 指标面板。
