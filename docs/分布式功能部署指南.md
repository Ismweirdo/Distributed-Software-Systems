# 分布式功能部署指南

本文档基于当前仓库代码，给出可落地的部署方式，分为：

- 单实例部署（`8083`）
- 双实例 + Nginx（`8081` + `8082`）
- MySQL 主从读写分离 + Elasticsearch 商品搜索（推荐）
- 单实例部署（`8083`）
- 双实例 + Nginx（`8081` + `8082`）

## 1. 环境要求
- Java 17+
- Maven 3.6+
- MySQL 8+
- Redis 6+
- Nginx 1.20+
- 可选：Elasticsearch 7.x+

## 2. 一键启动（推荐）

仓库根目录已提供 `docker-compose.yml`，包含：

- MySQL 主库 `mysql-master`（宿主机 `3307`）
- MySQL 从库 `mysql-slave1`（宿主机 `3308`）
- MySQL 从库 `mysql-slave2`（宿主机 `3309`）
- MySQL 复制初始化任务 `mysql-replication-init`（自动配置主从复制）
- Redis（宿主机 `6379`）
- Elasticsearch（宿主机 `9200`）
- 后端服务 `seckill-service`（宿主机 `8084` -> 容器 `8083`）

启动命令：

```bash
docker compose up -d
```

主从复制状态检查：

```bash
docker exec seckill-mysql-slave1 mysql -uroot -p123456 -e "SHOW REPLICA STATUS\G"
docker exec seckill-mysql-slave2 mysql -uroot -p123456 -e "SHOW REPLICA STATUS\G"
```

预期关键字段：
- `Replica_IO_Running: Yes`
- `Replica_SQL_Running: Yes`

## 3. 初始化 MySQL（手工部署时）

### 2.1 创建数据库并导入商品表

```sql
CREATE DATABASE IF NOT EXISTS seckill_db DEFAULT CHARSET utf8mb4;
USE seckill_db;
SOURCE D:/CreatorProject/seckill-system/src/main/resources/db/init_seckill_product.sql;
```

### 2.2 创建用户表（必须手动执行）

```sql
CREATE TABLE IF NOT EXISTS `user` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `username` VARCHAR(64) NOT NULL,
  `password` VARCHAR(128) NOT NULL,
  `phone` VARCHAR(20) DEFAULT NULL,
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 4. 启动 Redis

```bash
redis-server
```

## 5. 部署方式 A：单实例（开发默认）

### 4.1 启动

```bash
mvn clean package -DskipTests
java -jar target/seckill-system-0.0.1-SNAPSHOT.jar
```

### 4.2 默认地址
- 应用：`http://localhost:8083`
- 商品列表：`http://localhost:8083/api/products/list`
- 登录页：`http://localhost:8083/index.html`

## 6. 部署方式 B：双实例 + Nginx

### 5.1 启动两个实例

```bash
java -jar target/seckill-system-0.0.1-SNAPSHOT.jar --spring.profiles.active=instance1
java -jar target/seckill-system-0.0.1-SNAPSHOT.jar --spring.profiles.active=instance2
```

实例端口：
- `instance1` -> `8081`
- `instance2` -> `8082`

### 5.2 配置 Nginx
使用仓库根目录 `nginx.conf`，重点确认：

- `upstream` 指向 `127.0.0.1:8081`、`127.0.0.1:8082`
- `location /api/` 使用 `proxy_pass http://seckill-backend-roundrobin;`
- `root` 指向本机静态资源目录：
  `D:/CreatorProject/seckill-system/src/main/resources/static`

### 5.3 验证
- 访问：`http://localhost/index.html`
- 调用：`http://localhost/api/products/list`
- 查看响应头：`X-Upstream-Server`（用于观察请求被分配到哪个实例）

## 7. Elasticsearch 商品搜索

当前默认配置为开启（`elasticsearch.enabled=true`，可通过环境变量覆盖）。

### 7.1 启用方式
在配置文件中设置：

```yaml
elasticsearch:
  enabled: true

spring:
  data:
    elasticsearch:
      repositories:
        enabled: true
      uris: http://127.0.0.1:9200
```

### 7.2 同步商品数据

```bash
curl -X POST http://localhost:8083/api/search/sync
```

### 7.3 验证搜索

```bash
curl "http://localhost:8083/api/search/keyword?keyword=手机&page=0&size=10"
```

## 8. 读写分离启用说明

代码中已经有：
- `@DataSourceSlave` 注解
- `DataSourceAspect` 切面（随机路由 `slave_1` / `slave_2`）

当前 `application.yml` 已包含主从动态数据源配置，默认端口如下：

```yaml
spring:
  datasource:
    dynamic:
      primary: master
      strict: false
      datasource:
        master:
          url: jdbc:mysql://127.0.0.1:3307/seckill_db
          username: root
          password: 123456
          driver-class-name: com.mysql.cj.jdbc.Driver
        slave_1:
          url: jdbc:mysql://127.0.0.1:3308/seckill_db
          username: root
          password: 123456
          driver-class-name: com.mysql.cj.jdbc.Driver
        slave_2:
          url: jdbc:mysql://127.0.0.1:3309/seckill_db
          username: root
          password: 123456
          driver-class-name: com.mysql.cj.jdbc.Driver
```

代码中读请求（如 `listProducts`、`getProductById`）标注了 `@DataSourceSlave`，通过切面随机路由到 `slave_1`/`slave_2`；写请求默认走 `master`。

可通过测试验证：

```bash
mvn test -Dtest=ReadWriteSeparationTest,SearchServiceTest
```

## 9. 常见问题

### 9.1 登录/注册报表不存在
请确认已手动创建 `user` 表。

### 9.2 商品接口可用但搜索接口无结果
请先执行 `POST /api/search/sync` 同步商品到 ES，再执行搜索。

### 9.3 双实例可访问但 Nginx 打不开页面
优先检查 `nginx.conf` 中 `root` 路径是否为你本机的真实项目路径。
