# 分布式功能部署指南

本文档基于当前仓库代码，给出可落地的部署方式，分为：

- 单实例部署（`8083`）
- 双实例 + Nginx（`8081` + `8082`）
- 可选启用 Elasticsearch

## 1. 环境要求
- Java 17+
- Maven 3.6+
- MySQL 8+
- Redis 6+
- Nginx 1.20+
- 可选：Elasticsearch 7.x+

## 2. 初始化 MySQL

### 2.1 创建数据库并导入商品表

```sql
CREATE DATABASE IF NOT EXISTS seckill_db DEFAULT CHARSET utf8mb4;
USE seckill_db;
SOURCE D:/CreatorProject/seckill-system/src/main/resources/db/init_seckill_product.sql;
```

### 2.2 创建用户表（必须手动执行）

```sql
CREATE TABLE IF NOT EXISTS `user` (
  `id` BIGINT NOT NULL AUTO_INCREMENT,
  `username` VARCHAR(64) NOT NULL,
  `password` VARCHAR(128) NOT NULL,
  `phone` VARCHAR(20) DEFAULT NULL,
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP,
  `update_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 3. 启动 Redis

```bash
redis-server
```

## 4. 部署方式 A：单实例（开发默认）

### 4.1 启动

```bash
mvn clean package -DskipTests
java -jar target/seckill-system-0.0.1-SNAPSHOT.jar
```

### 4.2 默认地址
- 应用：`http://localhost:8083`
- 商品列表：`http://localhost:8083/api/products/list`
- 登录页：`http://localhost:8083/index.html`

## 5. 部署方式 B：双实例 + Nginx

### 5.1 启动两个实例

```bash
java -jar target/seckill-system-0.0.1-SNAPSHOT.jar --spring.profiles.active=instance1
java -jar target/seckill-system-0.0.1-SNAPSHOT.jar --spring.profiles.active=instance2
```

实例端口：
- `instance1` -> `8081`
- `instance2` -> `8082`

### 5.2 配置 Nginx
使用仓库根目录 `nginx.conf`，重点确认：

- `upstream` 指向 `127.0.0.1:8081`、`127.0.0.1:8082`
- `location /api/` 使用 `proxy_pass http://seckill-backend-roundrobin;`
- `root` 指向本机静态资源目录：
  `D:/CreatorProject/seckill-system/src/main/resources/static`

### 5.3 验证
- 访问：`http://localhost/index.html`
- 调用：`http://localhost/api/products/list`
- 查看响应头：`X-Upstream-Server`（用于观察请求被分配到哪个实例）

## 6. Docker Compose 说明（可选）

仓库内 `docker-compose.yml` 包含 `mysql`、`redis`、`seckill-service`。

```bash
docker-compose up -d
```

注意：
- Compose 将 MySQL 映射到宿主机 `3307`（容器内仍是 `3306`）。
- 如果你不使用容器内后端，而是本机直接运行后端，请修改数据源 URL 端口为 `3307`。

## 7. Elasticsearch（可选）

当前默认配置为关闭（`elasticsearch.enabled=false`）。

### 7.1 启用方式
在配置文件中设置：

```yaml
elasticsearch:
  enabled: true

spring:
  data:
    elasticsearch:
      repositories:
        enabled: true
      uris: http://127.0.0.1:9200
```

### 7.2 同步商品数据

```bash
curl -X POST http://localhost:8083/api/search/sync
```

### 7.3 验证搜索

```bash
curl "http://localhost:8083/api/search/keyword?keyword=手机&page=0&size=10"
```

## 8. 读写分离启用说明（可选增强）

代码中已经有：
- `@DataSourceSlave` 注解
- `DataSourceAspect` 切面（随机路由 `slave_1` / `slave_2`）

但默认 `application.yml` 没有主从动态数据源配置。若要真正启用，请增加类似配置：

```yaml
spring:
  datasource:
    dynamic:
      primary: master
      strict: false
      datasource:
        master:
          url: jdbc:mysql://127.0.0.1:3306/seckill_db
          username: root
          password: 123456
          driver-class-name: com.mysql.cj.jdbc.Driver
        slave_1:
          url: jdbc:mysql://127.0.0.1:3306/seckill_db_slave1
          username: root
          password: 123456
          driver-class-name: com.mysql.cj.jdbc.Driver
        slave_2:
          url: jdbc:mysql://127.0.0.1:3306/seckill_db_slave2
          username: root
          password: 123456
          driver-class-name: com.mysql.cj.jdbc.Driver
```

## 9. 常见问题

### 9.1 登录/注册报表不存在
请确认已手动创建 `user` 表。

### 9.2 商品接口可用但搜索接口无结果
默认 ES 关闭，属于预期行为。按第 7 节启用后再同步数据。

### 9.3 双实例可访问但 Nginx 打不开页面
优先检查 `nginx.conf` 中 `root` 路径是否为你本机的真实项目路径。
