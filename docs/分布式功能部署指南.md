# 分布式功能部署指南

## 功能概述

本次更新实现了以下分布式系统核心功能：

1. **Redis 分布式缓存** - 商品详情页缓存，包含穿透/击穿/雪崩解决方案
2. **MySQL 读写分离** - 主从架构，读操作走从库，写操作走主库
3. **ElasticSearch 商品搜索** - 全文检索，支持关键词、价格区间等搜索

## 环境要求

- Java 17+
- MySQL 8.0+（需要配置主从复制）
- Redis 6.0+
- ElasticSearch 7.x+（需安装 IK 分词器）
- Maven 3.6+

## 部署步骤

### 1. MySQL 主从复制配置

#### 1.1 配置主库（Master）

编辑 `my.ini` (Windows) 或 `my.cnf` (Linux)：

```ini
[mysqld]
server-id=1
log-bin=mysql-bin
binlog-format=ROW
```

重启 MySQL 服务后，执行：

```sql
-- 创建同步用户
CREATE USER 'repl'@'%' IDENTIFIED BY '123456';
GRANT REPLICATION SLAVE ON *.* TO 'repl'@'%';
FLUSH PRIVILEGES;

-- 查看主库状态
SHOW MASTER STATUS;
-- 记录 File 和 Position 的值
```

#### 1.2 配置从库（Slave）

编辑 `my.ini` 或 `my.cnf`：

```ini
[mysqld]
server-id=2
relay-log=mysql-relay-bin
```

重启 MySQL 服务后，执行：

```sql
-- 配置主从关系
CHANGE MASTER TO
    MASTER_HOST='127.0.0.1',
    MASTER_USER='repl',
    MASTER_PASSWORD='123456',
    MASTER_LOG_FILE='mysql-bin.000001',  -- 替换为实际的 File
    MASTER_LOG_POS=154;                   -- 替换为实际的 Position

-- 启动同步
START SLAVE;

-- 检查同步状态
SHOW SLAVE STATUS\G
-- 确保 Slave_IO_Running 和 Slave_SQL_Running 都是 Yes
```

#### 1.3 创建数据库和表

在主库执行：

```sql
CREATE DATABASE seckill_db DEFAULT CHARSET utf8mb4;
USE seckill_db;
SOURCE D:/CreatorProject/seckill-system/src/main/resources/db/init_seckill_product.sql;
```

在从库执行（或者等待自动同步）：

```sql
CREATE DATABASE seckill_db_slave DEFAULT CHARSET utf8mb4;
-- 如果主从同步正常，这一步可以省略，数据会自动同步
```

### 2. Redis 安装与配置

#### Windows:

1. 下载：https://github.com/microsoftarchive/redis/releases
2. 解压后运行：
```cmd
redis-server.exe redis.windows.conf
```

#### Linux:

```bash
wget http://download.redis.io/releases/redis-6.2.6.tar.gz
tar xzf redis-6.2.6.tar.gz
cd redis-6.2.6
make
sudo make install
redis-server &
```

验证：
```bash
redis-cli ping
# 应返回 PONG
```

### 3. ElasticSearch 安装与配置

#### Windows:

1. 下载：https://www.elastic.co/cn/downloads/elasticsearch
2. 解压后运行：
```cmd

```

#### Linux:

```bash
wget https://artifacts.elastic.co/downloads/elasticsearch/elasticsearch-7.17.0-linux-x86_64.tar.gz
tar -xzf elasticsearch-7.17.0-linux-x86_64.tar.gz
cd elasticsearch-7.17.0
./bin/elasticsearch
```

#### 安装 IK 分词器：


## 常见问题

### 1. 缓存穿透问题

**现象**: 大量请求查询不存在的数据，直接打到数据库

**解决方案**: 
- 布隆过滤器拦截
- 空值缓存（TTL 设置较短时间）

### 2. 缓存击穿问题

**现象**: 热点 key 过期瞬间，大量请求打到数据库

**解决方案**:
- 互斥锁（分布式锁）
- 双重检查锁机制

### 3. 缓存雪崩问题

**现象**: 大量 key 同时过期，导致数据库压力激增

**解决方案**:
- 随机 TTL（基础时间 + 随机时间）
- 永不过期（异步更新）

### 4. 读写分离数据延迟

**现象**: 写完数据后，从库读取不到最新数据

**解决方案**:
- 强制读主库（某些关键业务）
- 等待从库同步完成
- 使用中间件判断同步状态

### 5. ElasticSearch 连接失败

**检查项**:
- ES 服务是否启动：`curl http://localhost:9200`
- 端口是否正确（默认 9200）
- 是否启用了安全认证（需要配置用户名密码）
- IK 分词器是否正确安装

## 性能优化建议

### 1. Redis 优化

- 使用连接池
- 合理设置 TTL
- 使用 Pipeline 批量操作
- 启用持久化（RDB+AOF）

### 2. MySQL 优化

- 主库配置更高的硬件
- 从库可以配置多个（负载均衡）
- 开启慢查询日志
- 定期分析表和优化索引

### 3. ElasticSearch 优化

- 合理设计分片数
- 使用副本提高读取性能
- 定期 refresh 和 flush
- 监控 JVM 堆内存
